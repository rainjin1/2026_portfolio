⏰ 미션 알람 시스템 (Smart Mission Alarm)잠에서 깰 때까지 멈추지 않는, 하드웨어 기반 미션 수행형 스마트 알람Demo Video: 🎬 YouTube 시연 영상Source Code: 💻 GitHub Repository1. 프로젝트 개요목표: 단순 청각적 자극을 넘어, 사용자가 신체적·인지적 미션을 수행해야만 종료되는 알람 시스템을 구축하여 확실한 기상을 유도합니다.주요 기능:시간 설정: OLED, 가변저항, 스위치를 활용한 직관적인 타이머 세팅.멀티 미션: 알람음 연주와 동시에 진행되는 비동기 비밀번호 입력 미션.스누즈(Snooze): 제스처 센서를 이용한 최대 3회 연장 기능.거리 제한: PSD 적외선 센서로 일정 거리 내에서만 입력을 허용하는 물리적 제약.최종 종료: 소음 측정(샤우팅) 미션 수행 후 환경 정보(날짜, 온습도) 브리핑.2. 기술 스택구분상세 내용LanguageHardwarePYC Basic (Raspberry Pi 4), Soda OSSensorsOLED, Piezo Buzzer, PSD, Gesture, Sound, SHT20(온습도), Potentiometer, LEDsLibrariesthreading, pop (HAL), random, timeDev EnvironmentJupyterLab (Real-time Debugging)3. 시스템 아키텍처Setup Phase: 스위치를 활용한 시간 설정 UI 제공 (OLED).Wait Phase: 설정된 시간에 도달할 때까지 백그라운드 타이머 가동.Mission Phase (Core): 알람음 연주와 동시에 3가지 미션 병렬 수행.비밀번호 입력: PSD 센서로 적정 거리를 유지하며 OLED에 무작위로 점멸하는 4자리 번호 입력.스누즈(Snooze): 좌→우 제스처 인식 시 5분 연장 (최대 3회).LED 가이드: 스레드를 활용해 스누즈 가능 여부 및 입력 상태를 시각적으로 유도.Final Phase: 소음 센서 기반 "Shout" 미션 후 실시간 온습도 및 날짜 브리핑.4. 핵심 기술 포인트🚀 비동기 멀티태스킹 기반 UX 설계부저, 스위치, 제스처, LED 가이드를 4개의 독립적인 스레드로 병렬 실행하여 delay()로 인한 시스템 멈춤 현상을 해결했습니다.플래그(Flag) 기반 생명주기 관리를 통해 스레드 간 자원 충돌을 방지하고 메인 스레드와 동기화했습니다.🛠️ 실시간 센서 디버깅 및 최적화JupyterLab의 대화형 환경을 통해 각 센서의 로우 데이터(Raw Data)를 실시간 모니터링하였습니다.데이터 튀는 현상을 방지하기 위해 평균값 필터링 안전장치를 적용하여 임계값의 정확도를 높였습니다.5. 트러블 슈팅이슈 1: CPU 점유율 과부하 및 스레드 중복 실행원인: 무한 루프 내 스레드가 제어 없이 생성되거나 유휴 시간 부족.해결: while 루프 조건을 alarm_on으로 통일하고, 루프 최상단에 개별 조건문을 추가하여 불필요한 연산 시 즉시 continue 되도록 설계하여 부하를 최적화했습니다.이슈 2: OLED 렌더링 충돌 (PSD 피드백 vs 비밀번호 UI)원인: 여러 센서의 피드백이 한 번에 OLED에 출력되며 시각적 노이즈 발생.해결: 우선순위 기반 화면 갱신 로직을 도입하여 거리 경고(Too Close/Far)를 최우선으로 출력하도록 처리했습니다.6. 프로젝트 회고파이썬 멀티스레딩을 실제 하드웨어에 접목하며 변수 제어와 동기화 조건의 중요성을 체감했습니다. 소프트웨어와 달리 하드웨어는 병렬 동작 시 병목 현상이 즉각적으로 나타났으며, 이를 해결하기 위해 변수 켜고 끄는 타이밍을 더욱 정교하게 설계했습니다. 기획 단계에서의 체계적인 설계가 실제 개발 비용(시행착오)을 줄이는 핵심임을 깨달은 프로젝트였습니다.Contact & More infoYouTube: 시연 영상 보기GitHub: 코드 상세 보기Email: [사용자 이메일 주소]
