컴퓨터 비전 기반 도형 터치 게임 (OpenCV Shape Touch Game)
1. 프로젝트 개요
목표: 웹캠을 통해 사용자의 움직임을 실시간으로 감지하고, 화면에 나타나는 동적 도형 객체와 상호작용하는 인터랙티브 게임 구현.

주요 기능:

실시간 움직임 감지: 차영상(Frame Difference) 알고리즘을 활용한 터치 인터랙션 구현.

객체 지향 설계: 공통 속성을 가진 부모 클래스(Figures)와 형태별 자식 클래스(Ball, Square, Triangle) 구조 설계.

동적 난이도 조절: 도형의 크기에 따른 점수 차등화 및 수명(Lifetime) 시스템 적용.

게임 로직: 시작 화면(도형 선택), 메인 게임(점수/제한 시간), 게임 오버 UI 제공.

2. 기술 스택
프로그램 언어: Python

라이브러리:

OpenCV (cv2): 영상 처리, 프레임 캡처, 도형 렌더링 및 움직임 감지.

NumPy: 도형의 회전 및 좌표 계산을 위한 행렬 연산.

Math/Random/Time: 부채꼴 생성 좌표 계산, 확률 기반 객체 생성, 프레임 타임 관리.

주요 알고리즘:

Frame Differencing: 이전 프레임과 현재 프레임의 차이를 분석하여 움직임 영역 추출.

Affine Transformation: 행렬 연산을 이용한 다각형 객체의 실시간 회전 효과.

3. 시스템 아키텍처
A. Initialization Phase: 카메라 설정 및 해상도 최적화, 메뉴 객체 생성.

B. Selection Phase (Menu): 사용자가 특정 도형을 터치하면 해당 스타일로 게임 모드 결정.

C. Game Phase (Core):

Preprocessing: 가우시안 블러 및 그레이스케일 변환을 통한 노이즈 제거.

Motion Detection: 이진화(Threshold) 처리된 차영상에서 객체 영역 내 흰색 픽셀 비율 계산.

Object Management: 수명이 다하거나 터치된 객체를 리스트에서 실시간 관리.

D. Scoring & UI Phase: 실시간 점수 합산, 남은 시간 카운트다운 및 최종 결과 출력.

4. 핵심 기술 포인트
클래스 상속을 통한 렌더링 최적화

부모 클래스(Figures)에서 수명 주기, 색상 변화(Fading), 충돌 판정 로직을 공통 관리하여 코드 중복을 최소화했습니다.

각 자식 클래스는 draw 메서드만 오버라이딩하여 고유의 기하학적 형태를 렌더링합니다.

사용자 경험(UX)을 고려한 시각적 피드백

get_display_color 로직을 통해 도형이 소멸 직전까지 서서히 하얗게 변하는 시각적 힌트를 제공합니다.

18% 확률로 생성되는 '함정(Bad Figure)' 객체를 검은색으로 고정하여 전략적인 게임 플레이를 유도했습니다.

효율적인 충돌 판정 알고리즘

전체 프레임을 분석하는 대신, 각 도형 객체의 ROI(Region of Interest) 내에서만 countNonZero를 수행하여 CPU 부하를 방지했습니다.

5. 트러블 슈팅
문제 1: 프레임 간 노이즈로 인한 비정상적인 터치 판정

해결: 가우시안 블러 크기를 (15, 15)로 조정하여 미세한 노이즈를 억제하고, thresh 임계값을 30으로 설정하여 유의미한 움직임만 포착하도록 최적화했습니다.

문제 2: 도형이 화면 구석에 생성되어 상호작용이 어려운 현상

해결: get_safe_point 함수를 설계하여 화면 하단 중앙을 기점으로 한 부채꼴 영역 내에서만 객체가 생성되도록 좌표 범위를 제한했습니다.

6. 프로젝트 회고
OpenCV를 활용한 단순 영상 출력을 넘어, 수학적 좌표 변환과 객체 지향 프로그래밍을 결합해 완성도 있는 애플리케이션을 구축한 경험이었습니다. 특히 리스트 슬라이싱(shapes[:])을 통한 안전한 객체 삭제 루프 구현과 행렬 연산을 활용한 실시간 그래픽 회전 처리를 통해 효율적인 파이썬 코딩 기법을 익힐 수 있었습니다. 향후 Mediapipe를 활용한 고도화된 손가락 관절 추적 기능을 접목해 정교한 조작감을 구현해보고자 합니다.
